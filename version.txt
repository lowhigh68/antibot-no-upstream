  Problem: l7/rate/adaptive_limit.lua hard-banned ip on ip_rate>1500/60s
  (~25 req/s) with 1800s TTL, blind to identity diversity and overall score.
  Single user installing/testing a browser extension (Claude Code observed,
  but pattern applies to any extension that prefetches or AI-agent that
  navigates) bursts 1000-1500 requests during first 60s — exceeded threshold,
  whole public IP (CGNAT) banned for 30 min. Affected user's NAT peers too.

      identities seen in window < cfg.rate.ip_surge_distinct_min (3) → hard
      ban with cfg.rate.ip_surge_ban_ttl (300s, was 1800).
    - core/redis_pool.lua: new safe_scard() helper.
    - l7/rate/counter.lua: SADD rate:ids:<ip> per request (TTL=ttl.rate) so
      adaptive_limit can compute distinct-identity diversity cheaply.
    - intelligence/scoring/compute.lua: register ip_surge signal + weight 25.
